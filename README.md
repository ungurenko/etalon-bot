# 🎯 EtalonBot

Telegram-бот «Эталонная Версия» — стратегический ИИ-помощник для личностного развития. Ведёт клиента от Точки А к эталонной версии себя через онбординг, персональную стратегию и проактивные проверки прогресса.

## ✨ Что умеет бот

- **🗺️ Многосферный онбординг** — 45 вопросов по 9 сферам жизни (работа, здоровье, отношения, финансы, саморазвитие, досуг, окружение, духовность, быт)
- **🎙️ Голосовые сообщения** — распознавание речи через Whisper и ответ ИИ на естественном языке
- **🏆 Эталонная версия себя** — загрузка и структурирование образа идеального себя в 7 блоках: ценности, цели, принципы, привычки, окружение, достижения, образ жизни
- **📋 Персональная стратегия** — генерация поэтапного плана от текущего состояния к эталонной версии
- **🔔 Проактивные чекины** — автоматические проверки прогресса и напоминания по расписанию
- **💬 Свободный диалог** — поддержание разговора с учётом контекста клиента, его Точки А и Эталона
- **🖼️ Генерация образов** — создание визуализаций целей и эталонной версии

## 🛡️ Панель администратора

- **👥 Управление клиентами** — просмотр списка, смена статусов (active / pending / blocked)
- **📚 База знаний** — добавление и просмотр материалов для работы с клиентами
- **📢 Рассылки** — отправка сообщений выбранным клиентам или всем пользователям
- **🔍 Просмотр эталонов** — чтение и редактирование эталонных версий клиентов
- **⚙️ Настройки бота** — выбор LLM-модели, редактирование промптов и системных инструкций

## 🛠️ Технический стек

| Компонент | Технология |
|-----------|------------|
| Язык | Python 3.11+ |
| Telegram-фреймворк | aiogram 3.x |
| ORM / База данных | SQLAlchemy 2.0 + aiosqlite |
| LLM | OpenRouter API (настраиваемая модель) |
| Распознавание речи | Groq Whisper |
| Планировщик | APScheduler |
| Контейнеризация | Docker |
| Хостинг | Railway |

## 🏗️ Архитектура

```
etalon_bot/
├── bot.py              # Точка входа, регистрация роутеров и middleware
├── config.py           # Переменные окружения и настройки
├── database/           # SQLAlchemy модели, engine, CRUD-операции
├── handlers/           # Роутеры aiogram: старт, онбординг, чат, план,
│                       # настройки клиента, админ-панель
├── keyboards/          # Inline-клавиатуры для клиента и администратора
├── middlewares/        # Аутентификация, rate limit, логирование
├── services/           # LLM, Whisper, построение контекста, стратегия,
│                       # чекины, напоминания, генерация изображений
├── scheduler/          # APScheduler: фоновые задачи и чекины
└── data/               # init_questions.json — вопросы онбординга
```

## 🚀 Запуск локально

```bash
# 1. Клонировать репозиторий
git clone <repo-url>
cd etalon-bot

# 2. Создать виртуальное окружение
python -m venv venv
source venv/bin/activate  # macOS/Linux
# venv\Scripts\activate  # Windows

# 3. Установить зависимости
pip install -r requirements.txt

# 4. Создать файл .env
# BOT_TOKEN=your_telegram_bot_token
# ADMIN_IDS=123456789,987654321
# OPENROUTER_API_KEY=your_openrouter_key
# GROQ_API_KEY=your_groq_key
# DB_PATH=data/etalon_bot.db

# 5. Запустить бота
python -m etalon_bot.bot
```

## 🚂 Деплой на Railway

Бот деплоится через Docker builder в polling-режиме. SQLite база данных хранится на персистентном Volume, смонтированном на `/data/`.

```bash
# Привязать проект (один раз)
railway link

# Деплой
railway up --ci
```

Конфигурация: `Dockerfile` и `railway.json` в корне репозитория.
