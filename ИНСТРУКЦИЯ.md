# Бот «Эталонная Версия» — Инструкция по использованию

Стратегический ИИ-помощник для личностного развития. Бот ведёт клиента от текущего состояния (Точка А) к его эталонной версии через персональную анкету, индивидуальную стратегию и проактивные проверки прогресса.

# Часть 1. Для Дианы (администратор)

## Вход в админ-панель

Отправь боту команду `/admin` — откроется панель управления с кнопками:

| Кнопка | Что делает |
|--------|-----------|
| **Клиенты** | Список клиентов, управление статусами |
| **Эталонные версии** | Загрузка эталона для клиента |
| **База знаний** | Материалы, техники, ссылки |
| **Статистика** | Общая статистика по боту |
| **Рассылка** | Отправка сообщения всем активным клиентам |
| **Настройки** | Расписание чекинов, напоминания, обращения |

---

## Управление клиентами

### Как попадает новый клиент

1. Человек нажимает `/start` в боте
2. Бот присваивает ему статус **«Ожидает»** (pending)
3. Тебе приходит уведомление о новом пользователе
4. Ты заходишь в `/admin` → **Клиенты** и видишь его в списке

### Статусы клиентов

| Статус | Значение |
|--------|---------|
| **Ожидает** (pending) | Новый пользователь, ещё не активирован |
| **Активен** (active) | Может пользоваться ботом |
| **Неактивен** (inactive) | Деактивирован, бот не отвечает |
| **Заблокирован** (blocked) | Полная блокировка доступа |

### Действия с клиентом

Нажми на имя клиента в списке — откроется карточка с информацией и кнопками:

- **Активировать** — одобрить нового клиента (бот попросит ввести имя клиента для обращения)
- **Деактивировать** — временно отключить доступ
- **Заблокировать** — полностью закрыть доступ
- **Отклонить** — отклонить заявку (переводит в заблокированные)
- **Эталон** — загрузить эталонную версию для этого клиента
- **Стратегия** — сгенерировать стратегию
- **Точка А** — посмотреть все ответы клиента из анкеты (8 сфер)
- **Стратегия (просмотр)** — посмотреть сгенерированную стратегию

Список клиентов постраничный — по 5 на страницу, кнопками листается.

---

## Загрузка эталонной версии

Эталонная версия — это описание идеального состояния клиента по 7 блокам. Без неё нельзя сгенерировать стратегию.

### Способ 1: Свободная форма

1. Заходи в карточку клиента → **Эталон** → **Свободная форма**
2. Запиши голосовое или напиши текстом описание эталонной версии клиента
3. ИИ автоматически разобьёт текст на 7 блоков
4. Проверь превью, при необходимости отредактируй отдельные блоки
5. Нажми **Сохранить**

### Способ 2: По блокам

1. Заходи в карточку клиента → **Эталон** → **По блокам**
2. Бот последовательно попросит заполнить каждый из 7 блоков (текстом или голосом)
3. Проверь превью
4. Нажми **Сохранить**

### 7 блоков эталонной версии

1. **Ключевые способности и таланты** — в чём человек от природы силён
2. **Идеальная реализация** — как выглядит его идеальная деятельность
3. **Финансовая модель** — желаемый уровень дохода и его источники
4. **Отношения и окружение** — какие люди рядом, какие отношения
5. **Тело и энергия** — здоровье, физическая форма, энергия
6. **Внутреннее состояние** — эмоции, психологический комфорт
7. **Образ жизни и среда** — где живёт, как проходит день, привычки

При превью можно:
- **Сохранить** — записать в базу
- **Редактировать** — выбрать конкретный блок и переписать
- **Отмена** — отменить и начать заново

## Генерация стратегии

### Условия

Стратегию можно сгенерировать только когда:
- Клиент прошёл онбординг (заполнил анкету по 8 сферам)
- Загружена эталонная версия

### Процесс

1. Карточка клиента → **Генерация стратегии**
2. ИИ создаёт стратегию на основе Точки А (ответы анкеты) и Эталона
3. Стратегия разбивается на **этапы**, каждый этап содержит **конкретные задачи**
4. Появится превью с кнопками:
   - **Отправить клиенту** — сохранить и отправить (стратегия станет активной)
   - **Редактировать** — вручную поправить текст
   - **Перегенерировать** — создать заново с нуля

### После отправки

- Клиент получает уведомление о готовой стратегии
- В его меню появляется раздел «Мой план» с этапами и задачами
- Клиент может отмечать задачи как выполненные

---

## База знаний

Раздел для хранения полезных материалов, которые ИИ-помощник может использовать в ответах клиентам.

### Категории

| Категория | Что хранить |
|-----------|------------|
| **Техники против прокрастинации** | Методы борьбы с откладыванием |
| **Практики по сферам** | Упражнения для каждой из 8 сфер |
| **Ссылки на материалы** | Полезные ссылки, книги, курсы |

### Добавление материала

1. `/admin` → **База знаний** → **Добавить**
2. Введи название
3. Выбери категорию
4. Выбери сферу (1-8, необязательно)
5. Напиши содержание
6. Добавь ссылку (необязательно)

### Управление

- Просмотр по категориям с постраничной навигацией
- Каждый материал можно **активировать/деактивировать**, **редактировать** или **удалить**

---

## Рассылка

1. `/admin` → **Рассылка**
2. Напиши текст сообщения (поддерживается HTML-форматирование: `<b>жирный</b>`, `<i>курсив</i>`)
3. Бот покажет превью и количество активных клиентов
4. Нажми **Отправить** — сообщение уйдёт всем активным клиентам
5. После рассылки — отчёт: сколько доставлено, сколько ошибок

---

## Настройки

### Расписание чекинов

Бот автоматически отправляет клиентам проактивные вопросы о прогрессе.

- Формат: `пн,чт 10:00` (дни недели и время)
- По умолчанию: понедельник и четверг в 10:00 МСК
- Чекины получают только активные клиенты с активной стратегией

### Напоминания о неактивности

Бот отслеживает, когда клиент последний раз писал, и отправляет напоминания.

- Формат: `3 7 14` (три числа через пробел — дни)
- **Мягкое** (3 дня) — деликатное напоминание
- **Среднее** (7 дней) — более настойчивое
- **Жёсткое** (14 дней) — последнее предупреждение
- Проверка запускается ежедневно в 09:00 МСК

### Тёплые обращения

Слова, которые ИИ-помощник использует при обращении к клиентам.

- Формат: через запятую
- По умолчанию: `милая, невероятная, дорогая, прекрасная, чудесная`
- ИИ случайно выбирает одно из них для каждого ответа

# Часть 2. Что видит клиент

## Начало работы

1. Клиент нажимает `/start`
2. Бот приветствует и сообщает, что заявка отправлена администратору
3. Клиент ждёт активации (статус «Ожидает»)
4. После активации — бот предлагает начать анкету

## Команды клиента

| Команда | Что делает |
|---------|-----------|
| `/start` | Начало работы / возврат в главное меню |
| `/menu` | Открыть главное меню (или продолжить анкету, если не завершена) |
| `/help` | Справка о возможностях бота |

## Онбординг (анкета)

Анкета — это 45 вопросов по 8 сферам жизни. Создаёт «Точку А» — стартовое состояние клиента.

### 8 сфер

1. Здоровье и тело
2. Отношения
3. Карьера и бизнес
4. Финансы
5. Личностный рост
6. Эмоции и состояние
7. Окружение и среда
8. Смысл и миссия

### Как это работает

- Бот задаёт вопросы по очереди, сфера за сферой
- Отвечать можно **текстом** или **голосовым сообщением** (голос автоматически распознаётся)
- Можно отправлять ответ несколькими сообщениями подряд — бот подождёт 3 секунды тишины и объединит всё в один ответ
- Доступные кнопки во время анкеты:
  - **Пропустить** — пропустить вопрос
  - **Пауза** — приостановить, вернуться через `/menu`
  - **Начать сферу заново** — переотвечать все вопросы текущей сферы
- После завершения сферы — кнопка **Следующая сфера**
- После всех 8 сфер — предложение загрузить **фото** (необязательно, можно пропустить)
- Анкета сохраняется в базе данных — если бот перезапустится, прогресс не потеряется

## Главное меню

После завершения онбординга (или после получения стратегии) клиенту доступно меню:

| Кнопка | Что делает |
|--------|-----------|
| **Мой план** | Просмотр текущего этапа стратегии с задачами |
| **Мой прогресс** | Общий прогресс по всем этапам |
| **Написать помощнику** | Свободный чат с ИИ-помощником |
| **Практики** | Раздел практик (в разработке) |
| **Помощь** | Справочная информация |

## Работа с планом

- План разбит на **этапы**, в каждом — список задач
- Задачи можно отмечать галочкой (выполнено/не выполнено)
- Кнопки навигации: **Предыдущий этап** / **Следующий этап**
- Кнопка **Сохранить** — зафиксировать прогресс
- При выполнении всех задач этапа — автоматический переход к следующему

## Чат с ИИ-помощником

- Клиент нажимает **Написать помощнику** и просто пишет сообщение
- Принимаются **текстовые** и **голосовые** сообщения
- ИИ знает про Точку А клиента, его эталонную версию и текущий этап стратегии
- Ответы персонализированы — ИИ обращается тёплыми словами
- Бот ждёт 5 секунд тишины, чтобы объединить несколько сообщений в один запрос
- Длинные ответы автоматически разбиваются на части (лимит Telegram — 4096 символов)

## Проактивные сообщения

Бот сам пишет клиенту в двух случаях:

1. **Чекины по расписанию** — вопросы о прогрессе (по умолчанию пн и чт в 10:00)
2. **Напоминания при неактивности** — если клиент давно не писал (через 3, 7 и 14 дней)

# Часть 3. Для Алекса (техническая часть)

## Переменные окружения

| Переменная | Обязательность | Описание |
|-----------|---------------|---------|
| `BOT_TOKEN` | Обязательно | Токен Telegram-бота от @BotFather |
| `ADMIN_IDS` | Обязательно | ID администраторов через запятую (например `123456,789012`) |
| `OPENROUTER_API_KEY` | Обязательно | API-ключ OpenRouter для ИИ |
| `OPENROUTER_MODEL` | Нет | Модель ИИ (по умолчанию `google/gemma-4-31b-it:free`) |
| `GROQ_API_KEY` | Нет | API-ключ Groq для распознавания голоса (без него голосовые не работают) |
| `DB_PATH` | Нет | Путь к SQLite (по умолчанию `data/etalon_bot.db`) |
| `LOG_LEVEL` | Нет | Уровень логов (по умолчанию `INFO`) |
| `RATE_LIMIT_PER_MINUTE` | Нет | Лимит сообщений в минуту (по умолчанию `5`) |

## Запуск

```bash
# Локально
python -m etalon_bot.bot

# Docker
docker build -t etalon-bot etalon_bot/
docker run --env-file .env -v $(pwd)/data:/data etalon-bot
```

## Деплой

Деплоится на **Railway** с Volume `/data/` для персистентности SQLite.

## Структура проекта

```
etalon_bot/
  bot.py              — точка входа, регистрация роутеров
  config.py            — переменные окружения
  database/            — модели SQLAlchemy, engine, CRUD
  handlers/            — обработчики (admin, start, onboarding, chat, plan)
  keyboards/           — клавиатуры (admin_kb, client_kb)
  middlewares/         — авторизация, rate limit, логирование
  services/            — LLM (OpenRouter), Whisper (Groq), контекст, стратегия
  scheduler/           — APScheduler (чекины, напоминания)
  data/                — init_questions.json (45 вопросов)
```

## Порядок обработки сообщений

Роутеры зарегистрированы в порядке приоритета:
1. Админские хендлеры (только для ADMIN_IDS)
2. Онбординг (FSM-состояния)
3. План (просмотр стратегии)
4. Старт (команды `/start`, `/menu`, `/help`)
5. Чат (catch-all — ловит всё остальное, последний)

## Middleware-цепочка

1. **LoggingMiddleware** — логирование всех событий
2. **RateLimitMiddleware** — ограничение частоты сообщений
3. **AuthMiddleware** — проверка статуса пользователя, инъекция объекта `user` и `session`
